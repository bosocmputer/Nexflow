## คำอธิบายฟีเจอร์

### ฝั่งซื้อ

รองรับงานรับบิลซื้อจาก Email และ Shopee shipped email แล้วเตรียมข้อมูลเพื่อส่งเข้า SML เป็นใบสั่งซื้อ

ระบบช่วยอ่านข้อมูลจากอีเมลหรือไฟล์แนบ เช่น ชื่อผู้ขาย รายการสินค้า จำนวน ราคา วันที่เอกสาร เลขคำสั่งซื้อ และข้อมูลประกอบอื่น ๆ จากนั้นให้พนักงานตรวจรายการก่อนส่งเข้า SML

### ฝ่ายขาย

รองรับไฟล์ export จาก marketplace เช่น Shopee, Lazada, TikTok แล้วนำข้อมูลมาสร้างบิลขายใน BillFlow

พนักงานสามารถตรวจรายการสินค้า ปรับ mapping และเลือกว่าจะส่งเข้า SML เป็นใบสั่งขายหรือขายสินค้าและบริการ ตาม route ที่ตั้งค่าไว้

### Email IMAP รับบิล

เชื่อมต่อกล่องอีเมลของร้านผ่าน IMAP เพื่อดึงอีเมลและไฟล์แนบเข้าระบบอัตโนมัติ

ระบบรองรับหลาย inbox, ตั้งค่าคำกรองหัวข้อ, ผู้ส่งที่ยอมรับ, folder ที่อ่าน และดูผลการ poll ล่าสุดได้ เช่น พบกี่เมล ประมวลผลกี่เมล ข้ามเพราะอะไร

### Shopee Shipped Email เป็นบิลซื้อ

อ่านอีเมลจาก Shopee เช่น อีเมลแจ้งจัดส่งหรือยืนยันการชำระเงิน แล้วสร้างบิลซื้ออัตโนมัติ

ระบบเก็บเลขคำสั่งซื้อ ผู้ขาย วันที่เอกสาร รายการสินค้า ค่าส่ง ส่วนลด และหลักฐานอีเมลต้นฉบับ เพื่อให้ตรวจสอบย้อนหลังได้

### Marketplace Excel / CSV

รองรับไฟล์ export จาก Shopee, Lazada และ TikTok

ระบบจะ preview ก่อนสร้างบิล ตรวจ duplicate, group รายการตาม order, รวมรายการซ้ำ และสร้าง local bills เพื่อให้ตรวจสอบก่อนส่ง SML

### AI อ่านบิล / ไฟล์แนบ

ใช้ AI ช่วยอ่านข้อความจากอีเมล รูปภาพ PDF หรือไฟล์แนบ เพื่อดึงข้อมูลสำคัญออกมาเป็นโครงสร้างบิล

AI ไม่ได้ส่งเข้า SML ทันทีทุกกรณี ระบบยังมีขั้นตอน review เพื่อให้พนักงานตรวจรายการที่ไม่มั่นใจ ลดความเสี่ยงจากข้อมูลผิด

### ตรวจและแก้บิลก่อนส่ง SML

พนักงานสามารถเปิดบิลแต่ละใบเพื่อตรวจรายการสินค้า จำนวน หน่วย ราคา รหัสสินค้า SML และสถานะ mapping

ถ้าข้อมูลยังไม่ครบ เช่น ยังไม่ได้จับคู่สินค้า ยังไม่มีหน่วย หรือราคาผิด ระบบจะเตือนและป้องกันไม่ให้ส่งเข้า SML จนกว่าข้อมูลพร้อม

### Mapping สินค้า SML

ระบบช่วยจับคู่ชื่อสินค้าจากต้นทางกับรหัสสินค้าใน SML

เมื่อผู้ใช้ยืนยัน mapping แล้ว ระบบจะจำไว้และนำไปใช้กับบิลถัดไป ทำให้ไม่ต้องเลือกสินค้าเดิมซ้ำทุกครั้ง

### ส่ง SML ทีละใบ

เมื่อบิลพร้อมแล้ว พนักงานสามารถส่งเข้า SML ทีละใบ พร้อมเลือกค่าที่จำเป็น เช่น ผู้ขาย/ลูกค้า คลัง พื้นที่เก็บ VAT เวลาเอกสาร และเลขเอกสาร

หาก SML ส่งไม่สำเร็จ ระบบเก็บ error, payload, response และประวัติไว้ให้ตรวจสอบย้อนหลัง

### Bulk Send เข้า SML

สำหรับแพ็กเกจ Standard ขึ้นไป ระบบรองรับการส่งหลายบิลเข้า SML แบบ job

ผู้ใช้สามารถตรวจ preview ก่อนส่ง ดู progress ระหว่างส่ง ปิด dialog แล้วกลับมาดูต่อได้ และ retry เฉพาะรายการที่ส่งไม่สำเร็จ

### Dashboard / Logs

Dashboard แสดงภาพรวมงานที่ต้องจัดการ เช่น บิลรอตรวจ บิลพร้อมส่ง บิลส่งล้มเหลว หรือปัญหา email/SML

Logs เก็บประวัติการทำงาน เช่น ใครส่งบิลเมื่อไร SML ตอบอะไร error เกิดจากอะไร และควรแก้โดยผู้ใช้หรือทีม support

### ตั้งค่า SML Route / Doc Format / คลัง / VAT

กำหนดว่าแต่ละช่องทางจะส่งเข้า SML เมนูไหน เช่น ใบสั่งซื้อ ใบสั่งขาย หรือขายสินค้าและบริการ

ระบบรองรับการตั้งค่า doc format, prefix เลขเอกสาร, รูปแบบเลขรัน, คลัง, พื้นที่เก็บ, VAT และเวลาเอกสาร เพื่อลดการกรอกซ้ำตอนส่งบิล

### Shopee API Direct

สำหรับแพ็กเกจ Pro ขึ้นไป รองรับการเชื่อม Shopee Open Platform เพื่อดึง order จาก API โดยตรง

ระบบยังคงใช้แนวทาง review-first คือดึงข้อมูลมา preview และสร้าง local bills ก่อน ไม่ส่ง SML ทันที เพื่อให้ตรวจสอบได้ก่อนใช้งานจริง

### LINE แจ้ง Error

ระบบสามารถแจ้งเตือน admin ผ่าน LINE เมื่อเกิด error สำคัญ เช่น Email poll ล้มเหลวต่อเนื่อง, SML ส่งไม่สำเร็จ, token มีปัญหา หรือระบบต้องการให้ทีมดูแลตรวจสอบ

### LINE OA Inbox / Chat

สำหรับแพ็กเกจ Business ระบบรวมข้อความจาก LINE OA เข้า inbox ใน BillFlow

ทีมแอดมินสามารถตอบลูกค้า เก็บประวัติสนทนา ดูรูป/ไฟล์/audio จากลูกค้า และสร้างบิลจากข้อความหรือ media ใน chat ได้

### Quick Replies / Notes / Tags

ช่วยให้ทีมแอดมินทำงานกับลูกค้าใน LINE ได้เร็วขึ้น

Quick replies ใช้ตอบข้อความซ้ำ ๆ, notes ใช้จดข้อมูลภายในที่ไม่ส่งให้ลูกค้า, tags ใช้จัดกลุ่มลูกค้า เช่น VIP, ขายส่ง, รอติดตาม หรือ spam

### Multi-user

รองรับผู้ใช้หลาย role เช่น admin, staff, viewer

Admin จัดการ settings และผู้ใช้ได้ Staff ทำงานบิลและส่ง SML ได้ Viewer ใช้ดูข้อมูลบางส่วนโดยจำกัดสิทธิ์

## Fair Usage

BillFlow ไม่จำกัดจำนวนบิลแบบ hard limit เพื่อให้ลูกค้าไม่ต้องกังวลว่าบิลเยอะแล้วค่าใช้จ่ายจะพุ่งทันที

อย่างไรก็ตาม หากมีปริมาณการใช้งานสูงผิดปกติจนกระทบ performance, ค่า AI, storage, หรือ server resource ทีมงานจะคุยกับลูกค้าเพื่อปรับแพ็กเกจหรือแยก instance ให้เหมาะสม

ตัวอย่างกรณีที่อาจต้องปรับแพ็กเกจ:

- มีหลายบริษัท SML หรือหลายฐานข้อมูล SML
- มีหลายสาขาและต้องแยก workflow
- มีบิลจำนวนมากผิดปกติเมื่อเทียบกับร้านทั่วไป
- ต้องการ custom rule เฉพาะร้าน
- ต้องการ report เฉพาะทาง
- ต้องการ SLA หรือเวลาตอบสนองเฉพาะ

## ข้อความสำหรับนำเสนอ

BillFlow ไม่ได้มาแทน SML แต่ช่วยลดงานคีย์ซ้ำจาก Email, Shopee, Lazada, TikTok และส่งข้อมูลเข้า SML ได้เร็วขึ้น

ระบบช่วยให้พนักงานตรวจเฉพาะรายการที่ต้องแก้ เช่น สินค้าไม่ตรง หน่วยไม่ตรง ราคาไม่ครบ หรือข้อมูลจากต้นทางไม่ชัดเจน แทนการคีย์ทุกบิลใหม่ด้วยมือ

สำหรับลูกค้ารายแรก เราเสนอ Pilot 30 วัน เพื่อเริ่มใช้งานจริงด้วยต้นทุนต่ำ ถ้าระบบช่วยลดงานคีย์ได้จริง ลูกค้าค่อยเลือกแพ็กเกจใช้งานต่อหลังจบ Pilot

## หมายเหตุสำหรับตัวแทนขาย

- ราคานี้เป็นราคาขายแนะนำสำหรับลูกค้าปลายทาง
- ตัวแทนสามารถคิดค่า onsite training, data cleanup, หรือ custom setup เพิ่มได้ตามงานจริง
- ควรเสนอ Pilot ก่อน หากลูกค้ายังไม่มั่นใจ
- ถ้าลูกค้ามีปัญหาคีย์บิลชัดเจน เช่น วันละหลายสิบหรือหลายร้อยใบ ให้เสนอ Standard หรือ Pro
- ไม่ควรขายด้วยคำว่า OCR อย่างเดียว เพราะมูลค่าจริงคือ workflow เข้า SML, mapping สินค้า, ลดงานคีย์, ลด error และตรวจย้อนหลังได้
